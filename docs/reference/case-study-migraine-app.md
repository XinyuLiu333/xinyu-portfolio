## The Problem

Every migraine experience is unique. But the migraine impact on life is common.

**Patient emotions and quotes:**

- **Frustrated:** "I don't know what caused it, therefore I don't know how to stop it."
- **Hopeless:** "No matter what I do, it still feels hopeless."
- **Hurt:** "Sometimes it's easier to wish I am in a coma or worse rather than deal with it any longer."
- **Desperate:** "It's a never-ending feeling, because I never know when it will end."
- **Lonely:** "I never realized how hard migraines are on my professional and personal life until I met her. When I cannot go to work because of it, people will not understand and think 'you only have a headache.'"
- **FOMO:** "My migraine keeps me from doing the things I love most. There's so much to do, but I still can't do it."
- **Painkiller Stress:** "I always need more medication than I'm allowed to take."

At the heart, migraine sufferers desperately seek **Empowerment, Control, and Hope.**

> "A long time ago I resigned myself to the fact it was a hereditary thing and there's nothing I can do other than change my home life." — Missy

## Mapping Exercises

To gain a holistic understanding of the journey of someone living with migraines, the team started mapping activities and exploring opportunities.

### Migraine User Experience Map — Ongoing Migraine Attack

Multi-channel system (App, Watch, Voice UI) mapped across user journey:

**Ongoing Use:**
- User Goal: See quick takeaways when needed and learn more about migraine and triggers. Live life free without feeling overwhelmed.

**Pre Attack:**
- User Goal: Have predictability about migraine episodes so I can be aware and plan ahead. Take acute medication timely to maximize benefit.
- Receives trigger alert about conditions/events which might cause an attack
- Receives preventative med reminders
- Can log pre-migraine symptoms
- Receives preparation tips

**During Attack:**
- User Goal: Easily, discreetly, and most passively record basic information without making the attack worse.
- Option 1: Proactively log attack start
- Option 2: Smart prompt based on trigger alert, activity tracking, location, time, daily routine
- Migraine Mode activates: "Do Not Disturb," dark/low contrast screen, tailored features, passive context tracking
- Can log severity and acute meds from pre-picked personal list
- Can choose to input more details (ADLs, symptoms)

**Post Attack:**
- User Goal: Log this attack to help system look for patterns and provide personalized analysis, suggestions, and insights.
- Option 1: Proactively log end
- Option 2: Smart prompt based on activity tracking
- Attack recap: duration, severity, symptoms, acute meds, ADLs, context (location, weather), possible trigger events
- Trigger learnings: what the system learned about possible triggers
- Goal settings: personalize trigger tracking and goals to proactively prevent future attacks

### Migraine Interaction Map

Feedback loop system connecting:
- Triggers → Predictions → Migraine Attack
- Preventative Medications ↔ Prescription Adjustment
- Acute Medications ↔ Treatment Efficacy
- Quality of Life & Severity tracking
- Clinician Insights integration
- Trigger frequency/exposure reduction

## Behavior Design

Mapped behavioral techniques to gain deeper insight into patient motivations:

### Task Analysis Framework:
**People:**
- Knowledge/Skill Level vs Target Level
- Other Roles Involved
- Self-efficacy: Belief about Capability and Consequence
- Motivations: primary goal
- Existing Related Behaviors
- Other Ongoing Tasks

**Context/Environment:**
- Possible Distractions
- Triggers and Context
- Time of the Day
- Social/Professional Environment and Culture

**System:**
- Praise & Positive reinforcement
- Cognitive/Visual/Motor Loads: number of steps and clicks
- Time Estimated
- User Control Level
- Smart Defaults
- Triggers to Act
- Possible Errors: Humans and Systems
- Voice and Tone
- Messages: trainings, tips, timely feedback & feedforward
- Edge Cases

## Key Design Guidelines

- Making migraine entry effortless
- Adapting the system based on information acquired about each individual patient, context of use, and environment
- Supporting learning while experiencing

## Collecting Guidelines

- Build a patient-empowering relationship
- Explicitly communicate value
- Learning by doing
